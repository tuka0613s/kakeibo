# 爆速家計簿 機能仕様書

## コンセプト
「テンキーで金額入力 → カテゴリタップで即登録」を核とする、入力速度最優先の家計簿 PWA。

- **対象端末：** iPhone（iOS Safari）・iPad 11インチ（iOS Safari）
- **公開URL：** https://tuka0613s.github.io/kakeibo/

---

## 画面構成

### 1. 入力画面（メイン）

#### ヘッダーボタン

| ボタン | 機能 | 状態 |
|---|---|---|
| ⌨ 上 / 📋 下 | テンキーを上下に配置する Layout A/B を切り替え。設定は localStorage 永続保存 | ✅ |
| 0 右 / 0 下 | テンキーの 0 キー位置を切り替え（右端 ↔ 9 の真下）。設定は localStorage 永続保存 | ✅ |

**Layout A/B：**

| | Layout A（デフォルト） | Layout B |
|---|---|---|
| 順序 | 金額 → テンキー → カテゴリ → メモ → 履歴 | 金額 → メモ → 履歴 → カテゴリ → テンキー |
| 特徴 | 入力→タップの動線が短い | 履歴が見やすい |

ブラウザモード（ホーム画面未追加）の Layout B では、`hist-wrap` の `min-height` を解除してテンキーを常時表示する。
#### 各要素の仕様

| 要素 | 仕様 | 状態 |
|---|---|---|
| 金額表示 | 入力中はインディゴで強調。¥0 は薄色 | ✅ |
| テンキー | 0–9・⌫（1桁削除）・C（全消去）。右/下モード切替可。下モードでは00・−/+/= 追加 | ✅ |
| 予算進捗バー | 予算設定時のみ表示。緑→橙→赤で超過を視覚化 | ✅ |
| 支出/収入切替 | セグメントコントロール。カテゴリ一覧も連動 | ✅ |
| カテゴリ選択 | タップで即登録。金額0の場合はエラートースト。3行（12個）を超える場合は内部スクロール | ✅ |
| 日付表示 | 入力した日の日付と曜日を動的に表示（例：2026年5月1日 金曜日） | ✅ |
| メモ | 省略可。タップでボトムシートが開き入力（Enterキーで確定） | ✅ |
| 直近履歴 | 最新4件を1行サマリーで表示。タップで編集画面へ | ✅ |

### 2. カレンダー画面
ヘッダーの「0右/0下」ボタンで切替可能。設定は localStorage に永続保存。

**右モード（デフォルト）：**
```
7    8    9    ⌫
4    5    6    C
1    2    3    0
```

**下モード（calcスタイル）：**
```
7    8    9    +
4    5    6    −
1    2    3    =
0    00   ⌫    C
```

- `00`：2桁ゼロを追加（¥100・¥1,000 を素早く入力）
- `+` / `−`：第1オペランドを記憶して演算子を設定。金額表示に式（例: `1,000+`）を表示
- `=`：演算結果を金額に反映。カテゴリタップ時も自動評価してから登録
- `C` を最右端に配置（⌫ との押し間違いによる全消去を防止）

#### デフォルトカテゴリ

**支出（12件）：** 食費・外食・カフェ・交通費・日用品・医療・衣服・娯楽・通信費・光熱費・家賃・その他

**収入（6件）：** 給与・副業・ボーナス・投資・臨時収入・その他

### 3. 統計画面

#### 期間切り替え
ヘッダーのセグメントで月次 / 年次を切り替え。切り替えに連動してナビゲーションと全グラフが更新される。

| モード | ナビゲーション | サマリー対象 | 棒グラフ |
|---|---|---|---|
| 月次 | ‹ 年月 › で前後月に移動 | 選択月 | 選択月を末尾とした直近6ヶ月 |
| 年次 | ‹ 年 › で前後年に移動 | 選択年の合計 | 選択年の1〜12月 |

#### 各要素の仕様

| 要素 | 仕様 | 状態 |
|---|---|---|
| サマリー | 収入・支出・貯蓄を選択期間で動的計算 | ✅ |
| 推移グラフ | 収入/支出の縦棒グラフ（月次: 6ヶ月 / 年次: 12ヶ月） | ✅ |
| カテゴリ別支出 | 選択期間の支出上位8件を横バーチャートで表示 | ✅ |
| 貯蓄率 | プログレスバー + パーセント（収入に対する貯蓄割合） | ✅ |
| カテゴリ詳細 | カテゴリバーをタップで詳細オーバーレイを表示 | ✅ |

#### カテゴリ詳細オーバーレイ

カテゴリバーをタップすると全画面オーバーレイで以下を表示する。

| 要素 | 仕様 |
|---|---|
| サマリー | 合計・月平均・件数（月次：選択月のみ、年次：選択年全体） |
| 推移グラフ | 月次モードは直近6ヶ月、年次モードは選択年の12ヶ月 |
| 取引履歴 | 対象期間の取引を最大30件表示。タップで編集可。日付順 / 金額順の切替ボタンあり |

### 4. 取引編集画面

履歴またはカレンダーの取引をタップすると全画面で開く。

| 要素 | 仕様 | 状態 |
|---|---|---|
| ヘッダー | 左：キャンセル（テキスト）／中央：タイトル | ✅ |
| 支出/収入切替 | セグメントコントロール | ✅ |
| 金額入力 | カスタムテンキー（メイン画面と同一・iOS キーボード不使用）。ラベルは省略しタイトな配置 | ✅ |
| テンキー | 「00」ボタンを含む3行4列（または4行4列）。ボタン高さを縮小 | ✅ |
| カテゴリ | グリッド選択。2行（8個）を超える場合は内部スクロール。項目が少ない場合はタッチ微動防止 | ✅ |
| メモ | テキスト入力フィールド。ラベル省略 | ✅ |
| 日付 | date ピッカー。ラベル省略 | ✅ |
| フッター | 「削除」と「保存」ボタンを画面下部に固定して配置。余白を削減 | ✅ |

| データ | CSV エクスポート / インポート | エクスポート：今月 or 全件・BOM付き。インポート：CSV 追記・エラー行スキップ | ✅ |
| データ | デモデータを登録 | 2024〜2026年の約400件。既存データありの場合は確認シート経由 | ✅ |

| クラウド同期リスク | ⚠️ 競合シートを「キャンセル」して閉じた後にローカルで取引を登録すると、3秒後の自動バックアップが Drive の既存データを上書きする。競合が解決されるまで自動バックアップを止めるフラグは未実装 | 既知の制約 |
| アプリ情報 | アップデートを確認・強制更新 | キャッシュ削除 → SW解除 → 強制リロード | ✅ |

## 非機能要件

| 項目 | 要件 | 状態 |
|---|---|---|
| 対応環境 | iPhone / iOS Safari（PWA）・iPad 11インチ（iOS Safari） | ✅ 実機確認済み |
| オフライン | Service Worker でアセットをキャッシュ | ✅ |
| データ保存 | localStorage（単一デバイス内永続化）+ Google Drive（クラウド同期） | ✅ |
| 入力速度 | カテゴリタップで即登録・体感遅延ゼロ | ✅ |
| キーボード | カスタムテンキー使用・iOS デフォルトキーボードは出さない（メモのみ text input） | ✅ |
| レイアウト | iPhone は全画面・デスクトップ（1367px+）はモックフレーム・iPad（768〜1366px）はモックフレームなしフルスクリーン（横向きは「縦向きにしてください」オーバーレイ表示） | ✅ |
| Safe Area | Dynamic Island / ホームインジケーター対応 | ✅ |
| スクロール | `overscroll-behavior: none` でバウンスと親画面連動を防止 | ✅ |
| 固定パーツ | ヘッダー・フッターに `touch-action: none` を適用しズレを防止 | ✅ |
| ダブルタップ拡大 | `touch-action: manipulation` で全体無効化 | ✅ |
| 右端スワイプ誤ナビ | `history.pushState` で SPA 内に留める | ✅ |
